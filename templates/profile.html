System Briefing
Tracker Login

Der EifelLog Tracker benötigt eine aktive Anmeldung, damit Fahrten eindeutig deinem Fahrerprofil zugeordnet werden können.

  • Discord Login für normale Fahrer-Anmeldung.
  • Tracker Code als Fallback für lokale Tests.
  • Nach dem Login wird die Tracker-Oberfläche freigegeben.
  • Telemetrie startet erst nach erfolgreicher Authentifizierung.
Security Node

Der Client sendet keine Tourdaten, solange keine gültige Sitzung aktiv ist. Die finale Prüfung sollte immer serverseitig über deine Flask-API erfolgen.

  • Session Token lokal speichern.
  • API-Anfragen mit Bearer Token absichern.
  • Ungültige Sessions automatisch zurück zum Login schicken.
